# Warhammer Army Cost Calculator — Data Pipeline

This document explains how unit and kit pricing data is structured and generated.

The security sweep and future development **must not change this structure**.

---

# Data Philosophy

The application uses **static verified data**.

Rules:

• No estimates
• No scraped runtime data
• No external APIs
• No calculations stored in the dataset

All pricing and model counts represent **real Games Workshop retail kits**.

---

# Primary Dataset

The main dataset lives in:

```text
data/army-data-no-legends.json
```

This file contains **all factions and units**.

Structure:

```json
{
  "factions": {
    "space_marines": {
      "units": [
        {
          "id": "intercessor_squad",
          "name": "Intercessors",
          "points": 95,
          "models_per_box": 10,
          "box_price": 40
        }
      ]
    }
  }
}
```

Fields:

| Field          | Description                            |
| -------------- | -------------------------------------- |
| id             | Unique identifier                      |
| name           | Display name                           |
| points         | Warhammer unit points value            |
| models_per_box | Number of miniatures in the retail kit |
| box_price      | UK retail price of the kit             |

---

# Derived Dataset

When the build pipeline runs, faction datasets are generated in:

```text
data/factions/{faction-slug}/units.json
```

Example:

```text
data/factions/orks/units.json
```

Structure:

```json
{
  "faction": "Orks",
  "units": [...]
}
```

These files are what the **API serves to the frontend**.

---

# Kit Data

Two supporting datasets exist.

### Kit Mappings

```text
data/kit-mappings/{faction}.json
```

Maps units → retail kits.

Example:

```json
{
  "intercessor_squad": "intercessors",
  "captain": "space-marine-captain"
}
```

---

### Kit Definitions

```text
data/kits/{faction}.json
```

Defines retail kit details.

Example:

```json
{
  "intercessors": {
    "models": 10,
    "price": 40
  }
}
```

---

# Generation Scripts

The dataset is generated by Node scripts in:

```text
/scripts
```

The primary script used in the final pipeline:

```text
scripts/kit-data-generator.js
```

This script:

1. Reads the main dataset
2. Applies retail kit mappings
3. Generates faction-specific datasets
4. Outputs final files used by the API

---

# Final Data Flow

```text
army-data-no-legends.json
        ↓
kit-data-generator.js
        ↓
data/factions/{slug}/units.json
        ↓
API routes
        ↓
Frontend
```

---

# Important Rules

Future development **must follow these rules**.

### Do not:

• scrape prices dynamically
• fetch external APIs for pricing
• store user-generated data in datasets
• modify the dataset structure

### Allowed changes:

• update pricing
• add new units
• update points values
• add future factions

# Validation

The data pipeline enforces:

• every unit maps to a kit
• every kit has models + price
• no duplicate kit definitions
• valid JSON output

---

# Why Static Data

Static data ensures:

• fast load times
• no external dependencies
• predictable cost calculations
• simpler security model

The server **only serves static JSON files**.
